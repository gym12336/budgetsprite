# 个人记账系统「记账精灵 Web」详细设计文档

> **团队规模**：4人
> **开发周期**：3周
> **技术栈**：Vue 3 + Vite / ASP.NET Core 8 WebAPI / MySQL 8 / EF Core 8 / JWT

---

## 一、项目定位

一款面向个人用户的**网页版记账系统**，采用前后端分离架构，提供多维度可视化分析、预算管理、多账户管理、账单导入导出等功能，将"记账"变为高效的财务洞察工具。

### 核心价值
- **简洁**：3步快速记账，操作直觉化
- **清晰**：多维报表 + 图表分析，一眼看清收支
- **可控**：预算进度实时追踪，超支高亮预警
- **便携**：浏览器即用，无需安装，跨设备访问

## 二、产品功能设计

### 模块一：账单录入

| 子功能 | 说明 |
|---|---|
| 快速记账 | 页面浮动按钮，3步完成：选类型→填金额→选分类 |
| 完整记账 | 金额、二级分类、资金账户、日期时间、备注、标签、图片附件 |
| 账单导入 | 上传 CSV / Excel 文件，后端解析批量导入 |
| 周期账单 | 设置每月固定支出规则（如房租），系统按月自动生成 |
| 内部转账 | 不同资金账户间转账，不计入收支统计 |
| 记录编辑 | 支持编辑、删除、批量删除 |

### 模块二：分类管理

- **系统默认分类**：餐饮、交通、购物、娱乐、医疗、工资、副业等
- **二级分类**：餐饮 > 外卖/午餐/聚餐；收入 > 工资/兼职
- **自定义分类**：用户可新增/修改/删除分类，选择图标与颜色
- **分类类型**：每个分类区分"支出"或"收入"

### 模块三：可视化分析

- **月度总览**：本月收入、支出、结余，与上月同比变化
- **分类占比饼图**：各分类支出占比，支持点击查看明细列表
- **每日趋势图**：柱状图展示当月每天支出金额
- **月度趋势图**：折线图展示最近 12 个月收支走势
- **年度汇总**：全年分类支出排名
- 图表均使用 **ECharts** 渲染

### 模块四：预算管理

- 按月设置**总预算**和**分类子预算**
- 进度条实时显示已用/总额/剩余
- 超出 80% 时橙色警告，超出 100% 时红色高亮（页面内视觉警告，无需系统通知）
- 历史各月预算完成情况查询

### 模块五：资金账户管理

- 多账户类型：现金、储蓄卡、信用卡、支付宝、微信、其他
- 各账户余额展示，支持手动余额校正
- 首页展示全部账户余额汇总
- 账户间内部转账记录

### 模块六：用户认证

- 注册（用户名 + 密码）/ 登录
- JWT Bearer Token 鉴权，前端存 `localStorage`
- Refresh Token 机制，Token 过期自动续期
- 修改密码、修改昵称/头像

### 模块七：数据导出

- 按日期范围、分类、账户筛选后，一键导出 **Excel / CSV**
- 后端使用 **ClosedXML** 生成文件，前端触发下载
- 导出内容含：日期、类型、金额、分类、账户、备注、标签

## 三、技术架构

### 系统拓扑

```
┌─────────────────────────┐        ┌────────────────────────┐
│      浏览器（任意设备）    │  HTTPS │   ASP.NET Core 8       │
│                         │◄──────►│     WebAPI             │
│   Vue 3 + Vite          │        │                        │
│   Element Plus          │        │  ┌──────────────────┐  │
│   Pinia / Vue Router    │        │  │   MySQL 8         │  │
│   ECharts               │        │  │  (EF Core 8)      │  │
│   Axios                 │        │  └──────────────────┘  │
└─────────────────────────┘        └────────────────────────┘
```

### 前端分层

```
┌─────────────────────────────────────────────────┐
│  Pages / Views 层  (路由页面组件)                  │
│  - DashboardView / RecordsView / StatsView 等    │
├─────────────────────────────────────────────────┤
│  Components 层  (可复用 UI 组件)                  │
│  - RecordForm / BudgetProgress / ChartPanel      │
├─────────────────────────────────────────────────┤
│  Store 层  (Pinia 全局状态)                       │
│  - useAuthStore / useRecordStore / useBudgetStore│
├─────────────────────────────────────────────────┤
│  API 层  (Axios 封装)                            │
│  - request.js (拦截器 / Token注入 / 错误处理)     │
│  - api/records.js / api/stats.js 等              │
└─────────────────────────────────────────────────┘
```

### 后端分层（ASP.NET Core 8）

```
┌─────────────────────────────────────────────────┐
│  Controllers 层  (HTTP 路由 + 参数验证)           │
│  - RecordsController / StatsController 等        │
├─────────────────────────────────────────────────┤
│  Services 层  (业务逻辑)                         │
│  - RecordService / BudgetService / StatsService  │
│  - AccountService / CategoryService             │
├─────────────────────────────────────────────────┤
│  Repository 层  (数据访问)                       │
│  - EF Core DbContext + Repository 泛型基类       │
├─────────────────────────────────────────────────┤
│  Models 层  (领域模型 / DTO)                     │
│  - 实体类 + Request/Response DTO                 │
└─────────────────────────────────────────────────┘
```

### 核心设计模式

- **Repository 模式**：隔离数据访问层，方便单元测试
- **策略模式**：导出功能的多种格式（CSV / Excel）
- **观察者模式**：预算超支的状态响应（Pinia watch）
- **单例模式**：全局 Axios 实例、ECharts 主题配置
- **工厂模式**：ECharts 图表 option 的统一构建函数

## 十、风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|---|---|---|---|
| 前后端联调接口不一致 | 高 | 中 | 后端优先出 Swagger，前端先 Mock，联调期统一对齐 |
| ECharts 数据格式调试费时 | 中 | 低 | 前端先用静态假数据跑通图表，再替换真实接口 |
| LINQ 复杂聚合查询性能差 | 中 | 中 | 统计接口加缓存（MemoryCache），避免每次全表扫 |
| CSV/Excel 格式多样导致导入失败 | 中 | 中 | 提供模板下载，严格校验，错误行返回提示 |

## 十二、Git 协作规范

### 分支策略
- main：可发布版本，不直接推送
- dev：集成分支，功能开发完成后合并到此
- feature/xxx：个人功能分支
- hotfix/xxx：紧急修复

### 提交规范（Conventional Commits）
- feat: 添加账单导入功能
- fix: 修复预算进度计算错误
- refactor: 重构统计查询 LINQ
- docs: 更新 API 文档
- style: 调整表单间距

### Code Review
- 所有 PR 至少 1 人 review 后合并
- 大功能 PR 拆分为多个小 PR
